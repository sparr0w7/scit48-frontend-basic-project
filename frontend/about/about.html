로그인 없이, IP만으로 연결되는 익명 쪽지 네트워크

IpTalk은 접속 IP를 기반으로 사용자를 식별하는 실험적 메시지 프로젝트입니다. 데이터베이스 없이도 최소 단위로 연결되는 경험을 제공합니다.

보안 안내: 동일한 IP를 사용하는 사용자는 서로의 쪽지 내용을 볼 수 있습니다. 공용 네트워크(카페,학교,회사 등)에서는 민감한 정보 입력을 피하세요.

이 페이지는 추후 개발 예정인 플레이스홀더입니다.

어떻게 동작하나요?

Iptalk은 사용자의 IP를 기준으로 쪽지를 그룹화합니다. 같은 대역(/24)의 사용자는 서로 를 더 쉽게 탐색할 수 있으며, 동일 IP 내에서는 메시지가 공유될수 있습니다.

  1. 로그인, 회원가입 없음 ㅡ 접속 즉시 사용
  2. IP -> 익명 닉네임 매핑 (예: BlueFox-42)
  3. LocalStorage 기반 메모 흐름(서버 없는 체험)

빠른 시작

  • 1. 내 IP확인으로 현재 IP와 대역을 확인
  • 2. 쪽지쓰기에서 수신자 IP를 입력해 메시지 전송
  • 3. 쪽지함에서 받은 메시지 확인

※ 데모 단계에서는 동일 IP 사용자에게 메세지가 노출될 수 있습니다.

보안, 개인정보

  • 쪽지에 개인정보(이름, 전화번호, 주소, 계정)을 입력하지 마세요.
  • 공용 네트워크에서는 민감한 대화를 피해주세요.
  • IP는 변경될 수 있으며, 같은 사람ㅇ라도 다른 사용자로 인식될 수 있습니다.
  • 익명성은 완전하지 않습니다. 항상 주의하세요,

기술 스택

  • Frontend: HTML, CSS, JavaScript
  • 네트워크: ipify / ipapi 를 이용한 IP/지역 조회
  • 데이터: 데모 모드에서 LocalStorage 기반
